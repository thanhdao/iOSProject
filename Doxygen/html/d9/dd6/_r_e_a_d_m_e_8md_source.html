2 ## PDF Reader Core for iOS
3 
4 ### Introduction
5 
6 I've crafted this open source PDF reader code for fellow iOS
7 developers struggling with wrangling PDF files onto iOS device
8 screens.
9 
10 The code is universal and does not require any XIBs (as all UI
11 elements are code generated, allowing for greatest flexibility).
12 It runs on iPad, iPhone and iPod touch with iOS 4.0 and up. Also
13 supported are the Retina displays in all new devices and is ready
14 to be fully internationalized. The idea was to provide a complete
15 project template that you could start building from, or, just pull
16 the required files into an existing project to enable PDF
17 reading/viewing in your app(s).

24 After launching the sample app, tap on the left hand side of the
25 screen to go back a page. Tap on the right hand side to go to the
26 next page. You can also swipe left and right to change pages. Tap
27 on the screen to fade in the toolbar and page slider. Double-tap
28 with one finger (or pinch out) to zoom in. Double tap with two
29 fingers (or pinch in) to zoom out.
30 
31 This implementation has been tested with large PDF files (over
32 250MB in size and over 2800 pages in length) and with PDF files of
33 all flavors (from text only documents to graphics heavy magazines).
34 It also works rather well on older devices (such as the iPod touch
35 2nd generation and iPhone 3G) and takes advantage of the dual-core
36 processor (via CATiledLayer and multi-threading) in new devices.
37 
38 To see an example open source PDF Viewer App that uses this code
39 as its base, have a look at this project repository on GitHub:
40 https://github.com/vfr/Viewer

54 ### Notes
55 
56 Version 2.x of the PDF reader code was originally developed and
57 tested under Xcode 3.2.6, LLVM 1.7, iOS 4.3.5 and iOS 4.2.1 with
58 current development and testing under Xcode 4.5, LLVM 4.1, iOS 5/6.
59 Please note that as of v2.6, the code was refactored to use ARC.
60 
61 The overall PDF reader functionality is encapsulated in the
62 ReaderViewController class. To present a document with this class,
63 you first need to create a ReaderDocument object with the file path
64 to the PDF document and then initialize a new ReaderViewController
65 with this ReaderDocument object. The ReaderViewController class uses
66 a ReaderDocument object to store information about the document and
67 to keep track of document properties (thumb cache directory path,
68 bookmarks and the current page number for example).
69 
70 An initialized ReaderViewController can then be presented
71 modally, pushed onto a UINavigationController stack, placed in
72 a UITabBarController tab, or be used as a root view controller.
73 Please note that since ReaderViewController implements its own
74 toolbar, you need to hide the UINavigationController navigation
75 bar before pushing it and then show the navigation bar after
76 popping it. The ReaderDemoController class shows how this is
77 done with a bundled PDF file. To create a 'book as an app',
78 please see the ReaderBookDelegate class.

122 ### Compile Time Options
123 
124 In ReaderConstants.h the following #define options are available:
125 
126 `READER_BOOKMARKS` - If TRUE, enables page bookmark support.
127 
128 `READER_ENABLE_MAIL` - If TRUE, an email button is added to the toolbar
129 (if the device is properly configured for email support).
130 
131 `READER_ENABLE_PRINT` - If TRUE, a print button is added to the toolbar
132 (if printing is supported and available on the device).
133 
134 `READER_ENABLE_THUMBS` - If TRUE, a thumbs button is added to the toolbar
135 (enabling page thumbnail document navigation).
136 
137 `READER_DISABLE_IDLE` - If TRUE, the iOS idle timer is disabled while
138 viewing a document (beware of battery drain).
139 
140 `READER_SHOW_SHADOWS` - If TRUE, a shadow is shown around each page
141 and the page content is inset by a couple of extra points.
142 
143 `READER_STANDALONE` - If FALSE, a "Done" button is added to the toolbar
144 and the -dismissReaderViewController: delegate method is messaged when
145 it is tapped.
146 
147 `READER_DISABLE_RETINA` - If TRUE, sets the CATiledLayer contentScale
148 to 1.0f. This effectively disables retina support and results in
149 non-retina device rendering speeds on retina display devices at
150 the loss of retina display quality.
151 
152 `READER_ENABLE_PREVIEW` - If TRUE, a medium resolution page thumbnail
153 is displayed before the CATiledLayer starts to render the PDF page.
154 
155 ### ReaderDocument Archiving
156 
157 To change where the property list for ReaderDocument objects is stored
158 (~/Library/Application Support/ by default), see the +archiveFilePath:
159 method in the ReaderDocument.m source file. Archiving and unarchiving
160 of the ReaderDocument object for a document is mandatory since this is
161 where the current page number, bookmarks and directory of the document
162 page thumb cache is kept.
